## 🎯 새로운 모험의 시작

안녕하세요! 오늘은 제 블로그가 어떻게 **레벨업**했는지 이야기해보려 합니다. 마치 RPG 게임에서 새로운 직업으로 전직하는 것처럼, Ghost 블로그에서 Astro 정적 사이트로 완전히 갈아엎은 여정을 공유하겠습니다.

### 왜 이전을 결심했나? 🤔

Ghost로 블로그를 운영하면서 점점 느끼게 된 한계들이 있었습니다:

- **💰 호스팅 비용**: 월 $25-50의 지속적인 부담
- **🔧 커스터마이징 제약**: 원하는 디자인 구현의 어려움  
- **⚡ 성능 이슈**: 서버 응답 속도와 SEO 최적화 한계
- **🎨 창의적 표현**: 개발자로서 더 자유로운 표현 욕구

그러던 중 정적 사이트 생성기(SSG)의 매력에 빠지게 되었고, 특히 **Astro**의 Island Architecture와 뛰어난 성능에 매료되어 이전을 결심했습니다.

## ⚔️ 기술 스택 선택: 나의 파티 구성하기

RPG에서 파티원을 신중하게 선택하듯, 새로운 블로그의 기술 스택을 구성했습니다.

### 🏰 Main Tank: Astro 5.12

```typescript
// astro.config.mjs
import { defineConfig } from 'astro/config';
import mdx from '@astrojs/mdx';
import sitemap from '@astrojs/sitemap';

  site: 'https://luv20100918.github.io',

    }

```

**Astro를 선택한 이유:**
- 🚀 **Island Architecture**: 필요한 부분만 JavaScript 로드
- 📦 **Zero JS by default**: 기본적으로 순수 HTML 생성
- 🎯 **Content Collections**: 블로그 포스트 관리의 혁신
- ⚡ **빠른 빌드 속도**: 대용량 콘텐츠도 빠르게 처리

### 🛡️ Support: TypeScript + MDX

```typescript
// src/content/config.ts
import { defineCollection, z } from 'astro:content';

const blog = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    description: z.string(),
    pubDate: z.date(),
    updatedDate: z.date().optional(),
    heroImage: z.string().optional(),
    tags: z.array(z.string()).default([]),

```

- 🔍 컴파일 타임 에러 검출

- 📚 더 나은 코드 문서화

## 🗡️ 데이터 마이그레이션: 던전 공략하기

가장 까다로운 보스전이었습니다. Ghost의 모든 데이터를 Astro 형태로 변환하는 작업이었죠.

### Step 1: 데이터 백업 및 추출

```javascript
// migration/ghost-extractor.js
const fs = require('fs');
const path = require('path');

// Ghost JSON 데이터 파싱
const ghostData = JSON.parse(fs.readFileSync('/path/to/ghost-backup.json'));
const posts = ghostData.data.posts;

console.log(`총 ${posts.length}개의 포스트 발견`);
```

### Step 2: 콘텐츠 변환 스크립트

```javascript
// migration/convert-to-mdx.js
import { NodeHtmlMarkdown } from 'node-html-markdown';

const convertGhostToMDX = (ghostPost) => {
  const nhm = new NodeHtmlMarkdown();
  
  return {
    frontMatter: {
      title: ghostPost.title,
      description: ghostPost.custom_excerpt || ghostPost.excerpt,
      pubDate: new Date(ghostPost.published_at),
      slug: ghostPost.slug,
      tags: ghostPost.tags?.map(tag => tag.name) || []
    },
    content: nhm.translate(ghostPost.html)
  };
};

// 각 포스트를 MDX 파일로 생성
posts.forEach(post => {
  const converted = convertGhostToMDX(post);
  const fileName = `${converted.frontMatter.pubDate.toISOString().split('T')[0]}-${post.slug}.mdx`;
  
  const mdxContent = `---
${Object.entries(converted.frontMatter)
  .map(([key, value]) => `${key}: ${JSON.stringify(value)}`)
  .join('\n')}
---

${converted.content}
`;
  
  fs.writeFileSync(`src/content/blog/${fileName}`, mdxContent);
});
```

## 🤖 배포 자동화: 오토 파일럿 설정

GitHub Actions로 완전 자동화된 배포 파이프라인을 구축했습니다.

```yaml
# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'
          
      - name: Install dependencies
        run: npm ci
        
      - name: Build site
        run: npm run build
        
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

**자동화 효과:**
- ⚡ 푸시 후 3분 내 자동 배포
- 🔄 빌드 실패 시 자동 롤백
- 📧 배포 상태 슬랙 알림

```typescript
// 검색 컴포넌트 구현
import { useState, useEffect } from 'react';

export default function Search() {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState([]);

        type="text"
```
